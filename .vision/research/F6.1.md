# F6.1 — Open Source Documentation

## Overview

Public-facing project documentation for DX Terminal, covering positioning, quick start, architecture, contribution flow, and licensing.

## Scope

- README hero section with project name and clear tagline
- Feature overview for the operator cockpit, VDD workflow, runtime orchestration, and sync model
- Quick start path for local install and boot
- Architecture explanation for tmux, websocket, MCP gateway, and VDD integration
- Contribution and license hygiene for outside contributors

## Delivery Notes

- The documentation layer has to serve both local operators and future hosted dashboards.
- Shared guidance should live in `AGENTS.md`, with provider overlays extending it rather than forking it.
- The dashboard and repo docs should refer to the same execution model, not parallel descriptions.

## Acceptance Mapping

- README hero and tagline anchor the product story
- Feature list and quick start make the repo usable without prior context
- Architecture section explains how runtime panes, MCPs, and the dashboard stay aligned
- Screenshot/wiki links give a readable entry point for non-terminal collaborators
- `CONTRIBUTING.md` and `LICENSE` complete the open-source baseline

## Status: Done

Open-source documentation exists as a first-class project concern and is tracked in VDD alongside runtime and dashboard work.
