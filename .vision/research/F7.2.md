# F7.2 — 8-Framework QA Engine

## Overview

Phase-aware QA orchestration that evaluates work through multiple verification lenses and integrates with the VDD pipeline instead of running as a disconnected afterthought.

## Scope

- Multi-framework QA definition and reporting
- Hook integration so QA triggers at the right lifecycle boundary
- Verification-first stop conditions instead of silent task completion
- Metrics and evidence capture tied back into feature progress

## Architecture

- Hook layer detects test/verification moments and updates VDD state
- Dashboard surfaces QA posture through the project brief, feature readiness, and documentation sync health
- Acceptance criteria and evidence become the gate between `test` and `done`
- Automation assets and external MCP tools provide provider-neutral ways to inspect and verify behavior

## Acceptance Mapping

- Framework inventory must be explicit and inspectable
- Post-tool and stop hooks must keep QA in the loop before a feature is considered complete
- Delivery flow should read as discover → build → test → verify → ship
- QA runs should generate observable metrics and dashboard-visible evidence

## Status: Done

QA moved into the same delivery system as discovery, implementation, and verification, instead of living in a separate manual checklist.
