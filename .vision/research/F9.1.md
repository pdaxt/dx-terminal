# F9.1 — Dashboard Frontend-Backend Connectivity

## Problem Statement

The DX Terminal dashboard has sections that are disconnected from backend data:
- Vision section was hidden due to `MilestoneStatus::InProgress` parse error
- VDD research/discovery docs empty (no `.vision/research/*.md` files)
- Wiki page accessible but not navigable from dashboard context
- Pane context tab needed live tmux data (not just stale state)

## Root Cause Analysis

1. **Silent parse failures**: `load_vision()` used `.ok()` swallowing serde errors. A single missing enum variant (`InProgress` in `MilestoneStatus`) caused ALL vision data to fail silently.
2. **Project name mismatch**: Vision stored as "agentos" but directory is "dx-terminal" — API couldn't resolve.
3. **No research docs created**: VDD pipeline creates vision.json but research docs are created during feature spec phase — never done for existing features.

## Solution

1. Add `InProgress` to `MilestoneStatus` enum (done)
2. Add `tracing::warn` on parse errors instead of silent `.ok()` (done)
3. Fix project name to "dx-terminal" (done)
4. Create research docs for all features (in progress)
5. Audit all dashboard sections against API endpoints (in progress)
