# F3.1 — Vision-Driven Development

## Overview

Recursive vision tree with features, questions, decisions, tasks stored in `.vision/vision.json` per project.

## Architecture

- **Vision struct**: Project → Goals → Features → Tasks/Questions/Decisions
- **Storage**: Single JSON file at `.vision/vision.json` with optional sub-visions
- **API**: 12 MCP tools (vision_init, vision_tree, vision_drill, vision_work, etc.)
- **Web endpoints**: `/api/vision`, `/api/vision/tree`, `/api/vision/summary`, `/api/vision/list`

## Key Decisions

- **D3.1.1**: Store features in vision.json with optional sub-vision files (not separate files)
- Rationale: Keeps it simple. Sub-visions only when a feature is complex enough.

## Status: Done

All VDD MCP tools implemented and tested. Dashboard renders vision cockpit with expandable goal/feature tree.
